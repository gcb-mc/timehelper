# Timezone App

A single-page time zone lookup tool built entirely with HTML, CSS, and JavaScript — no frameworks, no build step. Created using GitHub Copilot Agent and Copilot CLI without writing any code manually.

---

## How It Works

The app is a self-contained `index.html` file with five screens:

1. **Welcome Screen** — Enter your name to get started. If you've visited before, the app recognizes you and offers to load your saved profile.
2. **Region Selection** — An interactive grid of continents (North America, South America, Europe, Africa, Asia, Oceania) displayed as clickable cards.
3. **Country Selection** — After picking a continent, choose your country from a scrollable grid.
4. **Time Zone Pick** — If your country has multiple time zones, pick the one that applies to you. Countries with a single zone skip this step automatically.
5. **Main App** — The core time zone tracker:
   - A searchable dropdown with all IANA time zones, grouped by region, with common-name aliases (e.g. "EST", "Pacific Time", "India", "Tokyo").
   - Add multiple time zones to a live-updating list showing current time, date, and UTC offset.
   - Star favorites and delete zones you no longer need.
   - Profile persistence via `localStorage` — your name, country, and home time zone are remembered across sessions.

## Prompts Used to Create This App

### Step 1 — Plan the App (Agent mode)

Started a chat in VS Code with the GitHub Copilot Agent and used **Plan** mode:

```
create another app in a new folder that is a simple time zone look up app with HTML, CSS and JavaScript. Include an input field to select time zones, a list to display them, and a delete button for each item.
```

I went back and forth with the agent for a bit, it asked me some questions, and eventually this is the additional prompting it used to upgrade and create the app:

```
The app should let the user enter their name, select a continent and country from an interactive map-style grid,
pick their time zone, and then land on a main screen where they can search and add multiple time zones
to a live-updating list. Include favorites, delete, and profile persistence using localStorage.
```

The Agent generated a plan outlining the file structure, screen flow, data model (continent → country → IANA zones), and UI components.

### Step 2 — Review and Accept the Plan

Reviewed the plan in VS Code, confirmed the screen layout and feature set, and selected **Keep** to accept the proposed file changes.

### Step 3 — Implement with Copilot CLI

Handed off the plan to **Copilot CLI** by selecting **Continue in Copilot CLI**. The CLI session:

- Scaffolded the full `index.html` with all five screens
- Built the `REGION_DATA` object mapping continents → countries → IANA time zones
- Created the searchable timezone dropdown with keyword aliases (city names, abbreviations like "PST", "CET", country names)
- Added `localStorage`-based profile management and favorites
- Styled everything with a dark theme using CSS custom properties

### Step 4 — Iterate and Refine

Used follow-up prompts in Agent mode to refine the app:

```
Add a welcome-back flow that detects returning users by name and lets them load their saved profile or start fresh.
```

```
Add common timezone aliases to the search so users can type things like "east coast", "hawaii", "india", or "london"
and find the right timezone.
```

```
Make the timezone dropdown keyboard-navigable with arrow keys and Enter to select.
```
